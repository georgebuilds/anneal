──────────────────── Interpretation ────────────────────────────────────────

Outcome: (b) Existing opts still don't move >L2 numbers.

The smem-tiling path (OptTile / OptTile+OptUpcast) adds zero throughput gain at any
tested size, including 2048³ and 4096³ where the working set exceeds M3's GPU L2.
This is the core finding.

The REASON is not "the smem path's data reuse isn't worth the overhead" but something
more fundamental: the bottleneck was never bandwidth (L2 or DRAM). The FMA probe in
the earlier diagnostic (TestDiag_FMAProbe) measured ~678 GFLOP/s pure ALU throughput.
The matmul sits at ~84 GFLOP/s — 12% ALU utilization. The flatness of throughput
across all sizes (512³–4096³) tells us the GPU is spending the same fixed fraction of
its ALU time on useful FMAs regardless of problem scale. That is a compute-scheduling
or occupancy problem, not a memory-access problem. Rearranging memory traffic via smem
tiling does not raise ALU utilization if ALU utilization is already the ceiling.

Implication for B3.5 (subgroup K-reduction):
B3.5 HAS a justified target. The gap to close is quantified:
  current:  84 GFLOP/s  (~12% of 678 GFLOP/s peak)
  headroom: 8× before reaching raw ALU ceiling
B3.5 subgroup intrinsics can reduce synchronization overhead in the reduction loop by
replacing workgroup-barrier-gated smem accumulation with warp-register accumulation
(subgroupAdd / subgroupShuffle). If barrier latency or smem round-trips are the reason
ALU utilization is stuck at 12%, subgroup ops directly address that bottleneck.
If not — if the root cause is something else (e.g. divergent memory addressing, too-few
wave fronts per SM, or a compiler artifact in the generated WGSL) — B3.5 will also
show flat numbers, which is itself informative: it would redirect attention to B4
(autotuned search over launch params) or B5 (Metal Performance Shaders integration).

Neither (a) ("opt already wins at >L2 sizes") nor (c) ("mixed/size-dependent") applies
here. The result is cleanly (b): the smem path does not pay off out of cache, and the
gap B3.5 would need to close is 8× on a fully quantified baseline.
