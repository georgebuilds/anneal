# B5 THREE-CONDITION STEP-LEVEL MEASUREMENT
# Methodology: 3 warmup + 7 timed iterations, median reported.
# Cond1: ANNEAL_BEAM unset, BeamDiskCacheReset() (identity, no opts).
# Cond2: ANNEAL_BEAM=1, BeamDiskCacheReset() (search runs, populates disk cache).
# Cond3: ANNEAL_BEAM unset, disk cache warm from Cond2 (cache-hit, opts applied).
# All three conditions run in the same process in TestB5_StepLevelBenchmark.
# MLP+Conv2d numbers from TestFusionBenchmark (tensor/nn/bench_test.go), Fusion OFF.
# ════════════════════════════════════════════════════════════════════════════════

## Workload: 1024×1024×1024 matmul (TestB5_StepLevelBenchmark)
  Cond1-ID     : median=32.661ms  min=32.249ms  max=33.558ms
  Cond2-Search : median=32.417ms  min=31.632ms  max=35.720ms
  Cond3-Cache  : median=32.435ms  min=31.594ms  max=33.070ms
  One-time search cost (warmup + 7 iters): 6.92s
  Cond3 vs Cond1: within noise (−0.7%)

## Workload: MLP (Batch=256, 1024-wide, 3 layers, 40 kernels, tensor/nn/bench_test.go)
  Cond1-ID     : median=383.73ms  min=380.05ms  max=387.06ms
  Cond2-Search : median=383.42ms  min=381.89ms  max=388.11ms
  Cond3-Cache  : median=382.75ms  min=380.54ms  max=387.08ms
  One-time search cost (embedded in 153s total Cond2 run vs 32s Cond1): ~121s
  Cond3 vs Cond1: within noise (−0.3%)

## Workload: Conv2d (Batch=16, 3→64, 64×64, 11 kernels, tensor/nn/bench_test.go)
  Cond1-ID     : median=702.27ms  min=691.20ms  max=713.57ms
  Cond2-Search : median=682.08ms  min=677.28ms  max=694.19ms
  Cond3-Cache  : median=682.08ms  min=673.65ms  max=689.28ms
  One-time search cost: embedded in Cond2 run (same run as MLP above)
  Cond3 vs Cond1: FASTER 2.9%  [Cond3 range 673–689ms fully below Cond1 min 691ms]

## Structural acceptance (all must hold):
  ✓ Default mode realize overhead: O(1) in-memory map lookup per kernel; no GPU work added.
  ✓ Cache invariants: Cond3 == Cond2 output (bit-identical verified by TestB5_CacheHitBitIdentical).
  ✓ Value-identity guard: fires on wrong WGSL hash (TestB5_SKCollisionGuard guard-A);
    passes on correct normalized hash (guard-B); SK-collision test constructed by
    injecting fake entry — genuine 64-bit SK collision impractical (probability ~2⁻⁶⁴).
  ✓ Full suite: 350 tests pass (uop/schedule/codegen/tensor/nn/backend/webgpu).

## Findings:
  - Large matmuls (1024³, MLP Batch=256) hit the scalar-WGSL ceiling; beam search finds
    identity as the winner. Cond3 step latency indistinguishable from Cond1.
  - Conv2d (small/dispatch-bound kernels) shows genuine 2.9% improvement. The beam
    search found a winning opt sequence (likely OptLocal) for the conv kernels. The
    Cond3 range (673–689ms) does not overlap the Cond1 range (691–714ms).
  - One-time search cost: ~7s for a single matmul kernel; ~121s for the full MLP+Conv2d
    kernel set. This is the upfront cost before the cache is warm.
  - Conclusion: B5 architecture is correct and conservative. For today's workloads the
    large-matmul case is latency-neutral; the small-kernel case picks up real wins.
    The cache and guard are production-ready.
